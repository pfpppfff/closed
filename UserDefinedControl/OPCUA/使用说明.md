# OPC UA 服务接口使用说明

## 概述
本文档介绍如何使用接口化的 OPC UA 通讯服务架构。新的架构支持依赖注入、单元测试、服务切换等高级功能，提供更灵活的使用方式。

## 主要组件说明

### 1. IOpcUaService 接口
- **用途**: 定义OPC UA通讯服务的标准接口
- **特点**: 支持依赖注入、便于单元测试、可扩展性强
- **位置**: `IOpcUaService.cs`

### 2. OpcUaGlobalService 实现类
- **用途**: IOpcUaService的真实实现，连接实际OPC UA服务器
- **特点**: 单例模式、线程安全、支持多种数据类型
- **位置**: `OpcUaGlobalService.cs`

### 3. MockOpcUaService 模拟实现
- **用途**: IOpcUaService的模拟实现，用于测试和开发
- **特点**: 内存中模拟数据、无需实际服务器、支持所有接口方法
- **位置**: `MockOpcUaService.cs`

### 4. OpcUaServiceManager 服务管理器
- **用途**: 管理当前活动的服务实例，支持服务切换
- **特点**: 全局访问点、支持依赖注入、线程安全
- **位置**: `OpcUaServiceManager.cs`

### 5. OpcUaServiceFactory 服务工厂
- **用途**: 创建不同类型的服务实例
- **特点**: 工厂模式、支持配置、环境自适应
- **位置**: `OpcUaServiceFactory.cs`

### 6. OpcUaHelper 辅助类
- **用途**: 提供简化的静态访问接口
- **特点**: 向后兼容、简单易用、基于服务管理器
- **位置**: `OpcUaHelper.cs`

## 服务接口使用方法

### 1. 基本服务创建和使用
```csharp
// 方式1: 使用默认服务
// OpcUaHelper 自动使用当前活动的服务实例
bool status = OpcUaHelper.ReadBool("1214.PLC1.System.Running");
OpcUaHelper.WriteBool("1214.PLC1.Control.Start", true);

// 方式2: 直接使用服务实例
IOpcUaService service = OpcUaServiceManager.Current;
bool status = service.ReadBool("1214.PLC1.System.Running");
service.WriteBool("1214.PLC1.Control.Start", true);

// 方式3: 创建特定类型的服务
IOpcUaService realService = OpcUaServiceFactory.CreateRealService();
IOpcUaService mockService = OpcUaServiceFactory.CreateMockService();
```

### 2. 服务切换和管理
```csharp
// 切换到模拟服务（用于测试）
var mockService = OpcUaServiceFactory.CreateMockService();
OpcUaServiceManager.SetService(mockService);

// 切换到真实服务
OpcUaServiceManager.ResetToDefault();

// 根据环境自动选择
#if DEBUG
    var service = OpcUaServiceFactory.CreateAutoService(isTestEnvironment: true);
#else
    var service = OpcUaServiceFactory.CreateAutoService(isTestEnvironment: false);
#endif
OpcUaServiceManager.SetService(service);
```

### 3. 在用户控件中使用依赖注入
```csharp
public partial class MyControl : UserControl
{
    private readonly IOpcUaService _opcService;
    
    // 支持依赖注入的构造函数
    public MyControl(IOpcUaService opcService = null)
    {
        _opcService = opcService ?? OpcUaServiceManager.Current;
        InitializeComponent();
    }
    
    private async void Button_Click(object sender, EventArgs e)
    {
        // 使用注入的服务
        bool success = await _opcService.WriteBoolAsync("nodeId", true);
    }
}

// 使用时：
// var control = new MyControl(); // 使用默认服务
// var control = new MyControl(mockService); // 使用指定服务
```

## 在用户控件中使用示例

### 按钮控件示例
```csharp
public partial class MyButtonControl : UserControl
{
    private string _nodeId = "1214.PLC1.Control.Button1";
    
    private void button_Click(object sender, EventArgs e)
    {
        // 切换按钮状态
        bool success = OpcUaHelper.ToggleBool(_nodeId);
        
        if (success)
        {
            // 更新UI显示
            UpdateButtonDisplay();
        }
    }
    
    private void UpdateButtonDisplay()
    {
        // 读取当前状态并更新显示
        bool currentState = OpcUaHelper.ReadBool(_nodeId);
        this.BackColor = currentState ? Color.Green : Color.Gray;
    }
    
    // 定时器更新状态
    private void timer_Tick(object sender, EventArgs e)
    {
        UpdateButtonDisplay();
    }
}
```

### 数值输入控件示例
```csharp
public partial class MyNumericControl : UserControl
{
    private string _nodeId = "1214.PLC1.Control.SetValue";
    
    private void textBox_KeyPress(object sender, KeyPressEventArgs e)
    {
        if (e.KeyChar == (char)Keys.Enter)
        {
            WriteValue();
        }
    }
    
    private void WriteValue()
    {
        string input = textBox.Text;
        
        if (OpcUaHelper.IsValidNumber(input))
        {
            bool success = OpcUaHelper.WriteFloatFromString(_nodeId, input);
            
            if (!success)
            {
                MessageBox.Show("写入失败！");
            }
        }
        else
        {
            MessageBox.Show("数据输入错误！");
        }
    }
    
    // 异步写入示例
    private async void WriteValueAsync()
    {
        string input = textBox.Text;
        
        if (OpcUaHelper.IsValidNumber(input))
        {
            float value = Convert.ToSingle(input);
            bool success = await OpcUaHelper.WriteFloatAsync(_nodeId, value);
            
            if (!success)
            {
                MessageBox.Show("写入失败！");
            }
        }
    }
}
```

## 连接状态监控

### 注册连接状态变化事件
```csharp
public partial class Form1 : Form
{
    private void Form1_Load(object sender, EventArgs e)
    {
        // 注册连接状态变化事件
        OpcUaHelper.RegisterConnectionStatusChanged(OnConnectionStatusChanged);
    }
    
    private void OnConnectionStatusChanged(object sender, bool isConnected)
    {
        // 在UI线程中更新状态显示
        this.Invoke(new Action(() =>
        {
            statusLabel.Text = isConnected ? "已连接" : "断开连接";
            statusLabel.ForeColor = isConnected ? Color.Green : Color.Red;
        }));
    }
    
    private void Form1_FormClosed(object sender, FormClosedEventArgs e)
    {
        // 取消注册事件
        OpcUaHelper.UnregisterConnectionStatusChanged(OnConnectionStatusChanged);
    }
}
```

## 总结

新的 OPC UA 全局单例通讯服务提供了以下优势：

1. **统一管理**: 全局单例确保连接的一致性
2. **线程安全**: 支持多线程环境使用
3. **简化接口**: OpcUaHelper 提供了简洁的使用方式
4. **类型安全**: 强类型方法避免类型转换错误
5. **异步支持**: 提供异步方法避免UI阻塞
6. **配置管理**: 灵活的配置管理机制
7. **错误处理**: 内置错误处理和恢复机制

通过使用这个服务，可以大大简化 OPC UA 通讯的实现和维护工作。
