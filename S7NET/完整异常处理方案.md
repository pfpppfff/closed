# PLC连接异常处理完整解决方案

## 问题总结

### 原始问题
用户断开PLC后，程序抛出多层嵌套异常：
- `S7.Net.PlcException`: "无法将数据写入传输连接: 远程主机强迫关闭了一个现有的连接"
- `SocketException`: 底层网络连接异常
- 异常调试窗口显示复杂的异常堆栈

### 问题根本原因
1. **异常识别不完整** - 没有正确识别S7.Net库的特定异常类型
2. **内部异常未处理** - 没有检查InnerException中的连接异常
3. **异常处理重复** - 内层和外层都处理异常，导致重复处理
4. **全局异常泄漏** - 某些异常可能逃逸到全局异常处理器

## 完整解决方案

### 1. 增强异常识别机制

#### 递归异常检查
```csharp
private bool IsConnectionLostException(Exception ex)
{
    if (ex == null) return false;

    // 检查当前异常
    if (CheckSingleException(ex))
        return true;

    // 递归检查所有内部异常
    var innerEx = ex.InnerException;
    while (innerEx != null)
    {
        if (CheckSingleException(innerEx))
            return true;
        innerEx = innerEx.InnerException;
    }

    return false;
}
```

#### 专门的异常类型检查
```csharp
private bool CheckSingleException(Exception ex)
{
    var message = ex.Message?.ToLower() ?? "";
    var typeName = ex.GetType().Name.ToLower();

    // S7.Net.PlcException专门处理
    if (typeName.Contains("plcexception"))
    {
        return message.Contains("无法将数据写入传输连接") ||
               message.Contains("远程主机强迫关闭了一个现有的连接");
    }

    // SocketException专门处理
    if (ex is System.Net.Sockets.SocketException socketEx)
    {
        return socketEx.SocketErrorCode == SocketError.ConnectionAborted ||
               socketEx.SocketErrorCode == SocketError.ConnectionReset;
    }

    // 其他连接异常...
}
```

### 2. 简化异常处理流程

#### 单层异常处理
```csharp
public async Task<T[]> ReadDBAsync<T>(int dbNumber, int startByte, int count = 1)
{
    if (!IsConnected)
        throw new InvalidOperationException("PLC未连接");

    try
    {
        return await Task.Run(() =>
        {
            if (!IsConnected)
                throw new InvalidOperationException("PLC连接已断开");

            // 直接执行读取，不再嵌套try-catch
            var result = _plc.Read(DataType.DataBlock, dbNumber, startByte, GetVarType<T>(), count);
            return ProcessResult(result);
        });
    }
    catch (Exception ex)
    {
        // 统一异常处理，包括内部异常检查
        if (IsConnectionLostException(ex) || IsConnectionLostException(ex.InnerException))
        {
            OnCommunicationError($"检测到PLC连接断开: {_ipAddress}");
            await ForceDisconnectAsync();
            OnConnectionStatusChanged(false);
            throw new InvalidOperationException("PLC连接已断开", ex);
        }
        else
        {
            OnCommunicationError($"读取DB块失败: {ex.Message}");
            throw;
        }
    }
}
```

### 4. 全局异常处理器

#### 最后防线
```csharp
// Program.cs
static void Main()
{
    // 设置全局异常处理器
    Application.SetUnhandledExceptionMode(UnhandledExceptionMode.CatchException);
    Application.ThreadException += Application_ThreadException;
    AppDomain.CurrentDomain.UnhandledException += CurrentDomain_UnhandledException;

    Application.EnableVisualStyles();
    Application.SetCompatibleTextRenderingDefault(false);
    Application.Run(new Form1());
}

private static void HandleException(Exception ex, string source)
{
    var message = "发生未处理的异常";
    
    // 检查是否是连接相关异常
    if (IsConnectionRelatedError(ex))
    {
        message = "PLC连接异常，程序将继续运行。\n\n如果问题持续，请检查PLC连接状态。";
    }

    MessageBox.Show(message, "系统异常", MessageBoxButtons.OK, MessageBoxIcon.Warning);
}
```
