# S7NET 写入功能使用说明

## 新增写入功能概述

按照您的要求，我已经添加了以下新的写入功能：

1. **写入Q点（输出点）bool值** - 支持批量和单个写入
2. **写入M区不同类型数据** - 支持多种数据类型的批量和单个写入
3. **写入M区位数据** - 支持位级别的批量写入

## 功能详细说明

### 1. 写入Q点（输出点）bool值

#### 1.1 批量写入Q点bool值
```csharp
/// <summary>
/// 写入Q点（输出点）bool量
/// </summary>
/// <param name="startByte">起始字节</param>
/// <param name="startBit">起始位</param>
/// <param name="values">bool值数组</param>
/// <param name="plcId">PLC ID（可选）</param>
/// <returns>写入是否成功</returns>
Task<bool> WriteQBitsAsync(int startByte, int startBit, bool[] values, string plcId = null)
```

**使用示例：**
```csharp
// 写入Q0.0开始的8个位
bool[] qValues = { true, false, true, true, false, false, true, false };
var success = await plcManager.WriteQBitsAsync(0, 0, qValues);

// 写入Q1.3开始的5个位
bool[] qValues2 = { true, true, false, true, false };
var success2 = await plcManager.WriteQBitsAsync(1, 3, qValues2);

// 指定PLC写入
var success3 = await plcManager.WriteQBitsAsync(0, 0, qValues, "PLC1");
```

#### 1.2 单个Q点bool值写入
```csharp
/// <summary>
/// 写入单个Q点bool值
/// </summary>
/// <param name="byteAddress">字节地址</param>
/// <param name="bitAddress">位地址</param>
/// <param name="value">bool值</param>
/// <param name="plcId">PLC ID（可选）</param>
/// <returns>写入是否成功</returns>
Task<bool> WriteQBitAsync(int byteAddress, int bitAddress, bool value, string plcId = null)
```

**使用示例：**
```csharp
// 写入Q0.5 = true
var success = await plcManager.WriteQBitAsync(0, 5, true);

// 写入Q2.3 = false
var success2 = await plcManager.WriteQBitAsync(2, 3, false);

// 指定PLC写入
var success3 = await plcManager.WriteQBitAsync(1, 7, true, "PLC2");
```

### 2. 写入M区不同类型数据

#### 2.1 批量写入M区数据
```csharp
/// <summary>
/// 写入M区不同类型数据
/// </summary>
/// <typeparam name="T">数据类型（byte, short, int, float, double等）</typeparam>
/// <param name="startByte">起始字节</param>
/// <param name="values">数据值数组</param>
/// <param name="plcId">PLC ID（可选）</param>
/// <returns>写入是否成功</returns>
Task<bool> WriteMAsync<T>(int startByte, T[] values, string plcId = null)
```

**支持的数据类型：**
- `byte` - 字节（8位）
- `short` - 短整型（16位）
- `int` - 整型（32位）
- `float` - 单精度浮点数（32位）
- `double` - 双精度浮点数（64位）
- `ushort` - 无符号短整型（16位）
- `uint` - 无符号整型（32位）

**使用示例：**
```csharp
// 写入M0开始的4个字节
byte[] mBytes = { 10, 20, 30, 40 };
var success = await plcManager.WriteMAsync<byte>(0, mBytes);

// 写入M10开始的3个短整型
short[] mShorts = { 1000, 2000, 3000 };
var success2 = await plcManager.WriteMAsync<short>(10, mShorts);

// 写入M20开始的2个整型
int[] mInts = { 100000, 200000 };
var success3 = await plcManager.WriteMAsync<int>(20, mInts);

// 写入M30开始的2个浮点数
float[] mFloats = { 3.14f, 2.71f };
var success4 = await plcManager.WriteMAsync<float>(30, mFloats);

// 写入M40开始的1个双精度浮点数
double[] mDoubles = { 123.456789 };
var success5 = await plcManager.WriteMAsync<double>(40, mDoubles);
```

#### 2.2 单个M区数据写入
```csharp
/// <summary>
/// 写入单个M区数据
/// </summary>
/// <typeparam name="T">数据类型</typeparam>
/// <param name="startByte">起始字节</param>
/// <param name="value">数据值</param>
/// <param name="plcId">PLC ID（可选）</param>
/// <returns>写入是否成功</returns>
Task<bool> WriteMSingleAsync<T>(int startByte, T value, string plcId = null)
```

**使用示例：**
```csharp
// 写入MB0 = 255
var success = await plcManager.WriteMSingleAsync<byte>(0, 255);

// 写入MW10 = 1234
var success2 = await plcManager.WriteMSingleAsync<short>(10, 1234);

// 写入MD20 = 987654
var success3 = await plcManager.WriteMSingleAsync<int>(20, 987654);

// 写入MD30 = 3.14159
var success4 = await plcManager.WriteMSingleAsync<float>(30, 3.14159f);

// 指定PLC写入
var success5 = await plcManager.WriteMSingleAsync<int>(50, 12345, "PLC1");
```

### 3. 写入M区位数据

```csharp
/// <summary>
/// 写入M区位数据
/// </summary>
/// <param name="startByte">起始字节</param>
/// <param name="startBit">起始位</param>
/// <param name="values">bool值数组</param>
/// <param name="plcId">PLC ID（可选）</param>
/// <returns>写入是否成功</returns>
Task<bool> WriteMBitsAsync(int startByte, int startBit, bool[] values, string plcId = null)
```

**使用示例：**
```csharp
// 写入M0.0开始的8个位
bool[] mBits = { true, false, true, true, false, false, true, false };
var success = await plcManager.WriteMBitsAsync(0, 0, mBits);

// 写入M5.3开始的10个位
bool[] mBits2 = { true, true, false, true, false, true, false, false, true, true };
var success2 = await plcManager.WriteMBitsAsync(5, 3, mBits2);

// 指定PLC写入
var success3 = await plcManager.WriteMBitsAsync(0, 0, mBits, "PLC2");
```

## 完整使用示例

```csharp
using S7NET.Services;
using System;
using System.Threading.Tasks;

class Program
{
    static async Task Main(string[] args)
    {
        var plcManager = MultiPlcServiceManager.Instance;
        
        try
        {
            // 初始化服务
            plcManager.Initialize();
            
            // 连接PLC
            await plcManager.ConnectAllPlcsAsync();
            
            // 1. 写入Q点数据
            Console.WriteLine("=== 写入Q点数据 ===");
            
            // 批量写入Q点
            bool[] qValues = { true, false, true, true, false, false, true, false };
            var qSuccess = await plcManager.WriteQBitsAsync(0, 0, qValues);
            Console.WriteLine($"批量写入Q0.0-Q0.7: {(qSuccess ? "成功" : "失败")}");
            
            // 单个写入Q点
            var qBitSuccess = await plcManager.WriteQBitAsync(1, 5, true);
            Console.WriteLine($"写入Q1.5=true: {(qBitSuccess ? "成功" : "失败")}");
            
            // 2. 写入M区数据
            Console.WriteLine("\n=== 写入M区数据 ===");
            
            // 写入字节数据
            byte[] mBytes = { 10, 20, 30, 40 };
            var mByteSuccess = await plcManager.WriteMAsync<byte>(0, mBytes);
            Console.WriteLine($"写入MB0-MB3: {(mByteSuccess ? "成功" : "失败")}");
            
            // 写入整型数据
            int[] mInts = { 100000, 200000 };
            var mIntSuccess = await plcManager.WriteMAsync<int>(10, mInts);
            Console.WriteLine($"写入MD10,MD14: {(mIntSuccess ? "成功" : "失败")}");
            
            // 写入单个浮点数
            var mFloatSuccess = await plcManager.WriteMSingleAsync<float>(20, 3.14159f);
            Console.WriteLine($"写入MD20=3.14159: {(mFloatSuccess ? "成功" : "失败")}");
            
            // 写入M区位数据
            bool[] mBits = { true, false, true, true, false, false, true, false };
            var mBitSuccess = await plcManager.WriteMBitsAsync(30, 0, mBits);
            Console.WriteLine($"写入M30.0-M30.7: {(mBitSuccess ? "成功" : "失败")}");
            
            Console.WriteLine("\n所有写入操作完成！");
            
        }
        catch (Exception ex)
        {
            Console.WriteLine($"操作失败: {ex.Message}");
        }
        finally
        {
            // 断开连接
            await plcManager.DisconnectAllPlcsAsync();
        }
    }
}
```

## 地址映射说明

### Q点地址映射
- `WriteQBitsAsync(0, 0, values)` → Q0.0 开始写入
- `WriteQBitsAsync(1, 3, values)` → Q1.3 开始写入（跨字节）
- `WriteQBitAsync(2, 5, value)` → Q2.5

### M区地址映射
- `WriteMAsync<byte>(0, values)` → MB0 开始写入
- `WriteMAsync<short>(10, values)` → MW10 开始写入
- `WriteMAsync<int>(20, values)` → MD20 开始写入
- `WriteMSingleAsync<float>(30, value)` → MD30
- `WriteMBitsAsync(0, 0, values)` → M0.0 开始写入

## 错误处理

所有写入方法都包含完整的错误处理机制：
- 连接断开自动检测
- 详细的错误日志
- 返回bool值表示写入是否成功

```csharp
try
{
    var success = await plcManager.WriteMSingleAsync<int>(0, 12345);
    if (success)
    {
        Console.WriteLine("写入成功");
    }
    else
    {
        Console.WriteLine("写入失败");
    }
}
catch (InvalidOperationException ex) when (ex.Message.Contains("PLC连接已断开"))
{
    Console.WriteLine("PLC连接断开，等待自动重连...");
    await Task.Delay(2000);
    // 可以选择重试
}
catch (Exception ex)
{
    Console.WriteLine($"写入失败: {ex.Message}");
}
```

## 注意事项

1. **数据类型对齐**: 确保写入的数据类型与PLC中的数据类型匹配
2. **地址范围**: 确保写入的地址在PLC的有效范围内
3. **连接状态**: 写入前确保PLC已连接
4. **数据长度**: 批量写入时注意数据长度不要超出PLC内存范围
5. **类型大小**: 不同数据类型占用的字节数不同，注意地址计算
6. **异常处理**: 建议使用try-catch处理可能的异常
7. **返回值检查**: 检查返回的bool值确认写入是否成功
