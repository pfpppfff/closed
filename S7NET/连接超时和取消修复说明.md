# PLC连接超时和取消功能修复说明

## 问题描述
当PLC不可达时，连接操作会长时间等待并最终超时，抛出异常：
```
S7.Net.PlcException: "Couldn't establish the connection to 192.168.2.10.
Message: 由于连接方在一段时间后没有正确答复或连接的主机没有反应，连接尝试失败。 192.168.2.10:102"
```

用户需要：
1. 快速检测连接失败（不要长时间等待）
2. 能够取消正在进行的连接操作
3. 优雅处理连接失败，不抛出异常

## 修复方案

### 1. 添加连接超时控制

在 `S7CommunicationService.cs` 中修改 `ConnectAsync` 方法：

```csharp
public async Task<bool> ConnectAsync(CancellationToken cancellationToken = default)
{
    // 设置连接超时时间为5秒
    using (var timeoutCts = new CancellationTokenSource(TimeSpan.FromSeconds(5)))
    using (var combinedCts = CancellationTokenSource.CreateLinkedTokenSource(cancellationToken, timeoutCts.Token))
    {
        await Task.Run(() =>
        {
            combinedCts.Token.ThrowIfCancellationRequested();
            _plc.Open();
            combinedCts.Token.ThrowIfCancellationRequested();
        }, combinedCts.Token);
    }
}
```

### 2. 添加连接失败检测

新增 `IsConnectionFailedException` 方法：

```csharp
private bool IsConnectionFailedException(Exception ex)
{
    var message = ex.Message?.ToLower() ?? "";
    
    return message.Contains("couldn't establish the connection") ||
           message.Contains("连接尝试失败") ||
           message.Contains("没有正确答复") ||
           message.Contains("主机没有反应") ||
           // 检查Socket错误码
           (ex is SocketException socketEx && 
            (socketEx.SocketErrorCode == SocketError.TimedOut ||
             socketEx.SocketErrorCode == SocketError.HostUnreachable));
}
```

### 3. 支持取消连接操作

#### 在连接管理器中添加取消支持：
- `ConnectPlcAsync(string plcId, CancellationToken cancellationToken)`
- `ConnectAllPlcsAsync(CancellationToken cancellationToken)`

#### 在界面中添加取消功能：
- 连接过程中按钮文本变为"取消连接"
- 点击可以立即取消连接操作
- 支持多PLC并行连接的取消

### 4. 优化用户界面

#### 连接按钮智能切换：
- **未连接时**: "连接所有PLC"
- **连接中**: "取消连接" (可点击取消)
- **连接完成**: "连接所有PLC"

#### 状态显示优化：
- 实时显示连接进度
- 清晰的超时和取消提示
- 友好的错误信息

## 修复效果

### ✅ 修复前的问题
- 连接超时需要等待很长时间（可能30秒以上）
- 无法取消正在进行的连接操作
- 连接失败时抛出异常，用户体验差
- 界面在连接过程中无响应

### ✅ 修复后的效果
- **快速超时检测** - 5秒内检测连接失败
- **可取消连接** - 随时可以取消连接操作
- **优雅异常处理** - 不抛出未处理异常
- **智能按钮状态** - 按钮文本和状态实时更新
- **并行连接支持** - 多个PLC同时连接，支持整体取消

## 使用方法

### 正常连接流程
1. 点击"连接所有PLC"按钮
2. 按钮变为"取消连接"，显示连接进度
3. 等待连接结果（最多5秒每个PLC）
4. 显示连接成功/失败结果

### 取消连接操作
1. 在连接过程中点击"取消连接"按钮
2. 立即取消所有正在进行的连接
3. 显示"连接已取消"状态
4. 按钮恢复为"连接所有PLC"

### 连接失败处理
1. PLC不可达时，5秒内检测到失败
2. 显示友好的错误信息："PLC 192.168.2.10 不可达或未响应"
3. 不抛出异常，程序继续正常运行
4. 可以重新尝试连接

## 技术特性

### 超时控制
- **连接超时**: 5秒（可配置）
- **快速失败**: 避免长时间等待
- **资源释放**: 自动清理超时的连接尝试

### 取消机制
- **CancellationToken**: 标准的.NET取消模式
- **组合取消**: 支持超时和手动取消
- **线程安全**: 多线程环境下的安全取消

### 异常处理
- **分类处理**: 区分超时、取消、连接失败等不同情况
- **友好提示**: 用户友好的错误信息
- **日志记录**: 详细的操作日志

### 界面响应
- **实时更新**: 按钮状态实时反映操作状态
- **非阻塞**: 界面始终保持响应
- **状态同步**: 连接状态与界面显示同步
