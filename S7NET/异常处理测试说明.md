# PLC连接断开异常处理修复说明

## 问题分析

### 原始问题
用户反馈断开PLC后，程序在`ReadDBAsync`方法中抛出异常：
```
S7.Net.PlcException:"无法将数据写入传输连接: 远程主机强迫关闭了一个现有的连接"
```

### 问题根本原因
1. **连接状态检查滞后** - `IsConnected`属性更新有延迟，实际连接已断开但状态还显示连接
2. **异常识别不完整** - `IsConnectionLostException`方法没有正确识别`S7.Net.PlcException`类型
3. **异常处理位置** - 异常在`_plc.Read()`调用时抛出，但处理逻辑在外层

## 修复方案

### 1. 增强异常识别
```csharp
private bool IsConnectionLostException(Exception ex)
{
    if (ex == null) return false;
    var message = ex.Message?.ToLower() ?? "";

    // 专门检查S7.Net.PlcException类型
    if (ex.GetType().Name == "PlcException")
    {
        return message.Contains("无法将数据写入传输连接") ||
               message.Contains("远程主机强迫关闭了一个现有的连接") ||
               message.Contains("unable to write data to the transport connection") ||
               message.Contains("connection was forcibly closed");
    }

    // 其他常见连接断开异常...
}
```

**改进点**：
- 专门检查`PlcException`类型
- 添加"无法将数据写入传输连接"关键字检查
- 支持中英文异常消息

### 2. 多层异常处理
```csharp
public async Task<T[]> ReadDBAsync<T>(int dbNumber, int startByte, int count = 1)
{
    // 第一层：基本连接状态检查
    if (!IsConnected)
    {
        throw new InvalidOperationException("PLC未连接");
    }

    try
    {
        return await Task.Run(() =>
        {
            // 第二层：读取前再次检查
            if (!IsConnected)
            {
                throw new InvalidOperationException("PLC连接已断开");
            }

            try
            {
                // 第三层：实际读取操作
                var result = _plc.Read(DataType.DataBlock, dbNumber, startByte, GetVarType<T>(), count);
                return ProcessResult(result);
            }
            catch (Exception readEx)
            {
                // 第四层：读取异常立即处理
                if (IsConnectionLostException(readEx))
                {
                    throw new InvalidOperationException("PLC连接已断开", readEx);
                }
                throw;
            }
        });
    }
    catch (Exception ex)
    {
        // 第五层：最终异常处理和状态更新
        if (IsConnectionLostException(ex))
        {
            OnCommunicationError($"检测到PLC连接断开: {_ipAddress}");
            await ForceDisconnectAsync();
            OnConnectionStatusChanged(false);
            throw new InvalidOperationException("PLC连接已断开", ex);
        }
        else
        {
            OnCommunicationError($"读取DB块失败 [DB{dbNumber}.DBX{startByte}]: {ex.Message}");
            throw;
        }
    }
}
```

**多层防护**：
1. **第一层** - 基本连接状态检查
2. **第二层** - 读取前再次检查
3. **第三层** - 实际读取操作
4. **第四层** - 读取异常立即处理
5. **第五层** - 最终异常处理和状态更新

### 3. 连接状态强制更新
```csharp
private async Task ForceDisconnectAsync()
{
    try
    {
        await Task.Run(() =>
        {
            try
            {
                _plc?.Close();
            }
            catch { }
        });
    }
    catch { }
}
```

**强制断开**：
- 立即调用`_plc.Close()`
- 忽略所有异常，确保执行完成
- 清理连接资源

## 修复效果

### ✅ 修复前的问题
- 抛出未处理的`S7.Net.PlcException`
- 程序可能崩溃或界面卡死
- 连接状态不更新
- 用户不知道发生了什么

### ✅ 修复后的效果
- **异常被正确捕获** - 不会抛出未处理异常
- **状态立即更新** - 连接状态立即变为断开
- **友好错误信息** - 显示"检测到PLC连接断开"
- **自动重连准备** - 触发自动重连机制

## 异常处理流程

### 正常情况
```
ReadDBAsync() → 检查连接 → 执行读取 → 返回数据
```

### 连接断开情况
```
ReadDBAsync() → 检查连接 → 执行读取 → 抛出PlcException
                                    ↓
                              识别为连接断开异常
                                    ↓
                              强制断开连接
                                    ↓
                              更新连接状态
                                    ↓
                              抛出友好异常
                                    ↓
                              上层捕获处理
                                    ↓
                              界面显示"连接断开，等待重连..."
```

## 技术细节

### 异常传播链
```
_plc.Read() → PlcException
     ↓
IsConnectionLostException() → true
     ↓
ForceDisconnectAsync() → 强制断开
     ↓
OnConnectionStatusChanged(false) → 状态更新
     ↓
throw InvalidOperationException("PLC连接已断开") → 友好异常
     ↓
上层catch → 界面更新
```

### 状态更新时序
```
1. 异常发生 → PlcException
2. 异常识别 → IsConnectionLostException = true
3. 强制断开 → ForceDisconnectAsync()
4. 状态更新 → OnConnectionStatusChanged(false)
5. 界面更新 → "连接断开，等待重连..."
6. 重连准备 → 心跳检测开始尝试重连
```
